프로젝트 개요

사용자가 자유롭게 자신만의 투자 포트폴리오를 구성, 분석 및 시뮬레이션할 수 있도록 설계된 동적 대화형 웹 애플리케이션입니다. 데이터 수집부터 시각화까지 전체 데이터 파이프라인을 자동화하여 사용자가 실시간으로 투자 전략을 테스트하고 개선할 수 있도록 지원합니다.

🎯

사용자 중심 설계

누구나 쉽게 투자 아이디어를 테스트하고 검증할 수 있는 직관적인 인터페이스.

🔄

데이터 자동화

수집, 처리, 분석, 시각화에 이르는 데이터 생명주기 전체를 자동화.

🧱

모듈형 아키텍처

유지보수와 향후 기능 확장을 용이하게 하는 견고하고 확장 가능한 설계.

시스템 아키텍처

안정성과 확장성을 보장하기 위해, 각 구성 요소가 명확하고 독립적인 역할을 수행하는 모듈형 아키텍처를 기반으로 프로젝트를 구축했습니다. 이러한 관심사 분리 원칙은 문제 해결 및 신규 기능 추가 효율성을 높이는 핵심이었습니다.

핵심 기능

대시보드는 초기 포트폴리오 구성부터 심층적인 성과 검증에 이르기까지 포괄적인 분석 경험을 제공하는 세 가지 주요 기능을 갖추고 있습니다.

동적 포트폴리오 구성

사용자는 기본 자산 목록에서 선택하거나 원하는 종목 티커를 직접 추가할 수 있습니다. 확장 가능한 메뉴와 5% 단위 슬라이더는 비중 할당을 직관적이고 정확하게 만듭니다.

실시간 성과 분석

비중이 조정될 때마다 연평균 수익률, 변동성, 샤프 지수와 같은 핵심 성과 지표(KPI)가 즉시 재계산되어 포트폴리오의 위험-수익 프로필에 대한 즉각적인 피드백을 제공합니다.

백테스팅 및 비교 분석

사용자의 포트폴리오를 S&P 500과 같은 시장 벤치마크와 비교하여 백테스트를 수행하며, 중요한 고점과 저점을 자동으로 강조 표시합니다. 또한 여러 포트폴리오를 저장하고 사용자 지정 기간 내에서 성과를 나란히 비교할 수 있습니다.

도전 과제 및 해결 방안

개발 과정에는 몇 가지 기술적인 장애물 극복이 포함되었습니다. 이러한 도전 과제들은 최종적으로 견고한 애플리케이션 아키텍처를 형성하는 데 중추적인 역할을 했습니다.

도전 1: 라이브러리 오류

문제:

데이터 소스에서 반환되는 일관성 없는 데이터 형식과 시각화 라이브러리의 엄격한 요구사항으로 인해 `KeyError` 및 `ValueError` 예외가 빈번하게 발생했습니다.

해결:

`data_fetcher.py`에 강력한 데이터 정제 계층을 구현했습니다. 이 계층은 컬럼 이름을 방어적으로 확인하고, 다양한 데이터 구조를 처리하며, 모든 데이터가 분석 엔진에 도달하기 전에 깨끗하게 정제되도록 보장합니다.

도전 2: 네트워크 보안

문제:

방화벽이 연결을 차단하여 `SSLError`가 발생하는 등 보안이 강화된 네트워크 환경(예: 회사)에서 데이터 수집이 실패했습니다.

해결:

`yfinance` 라이브러리의 최신 요구사항을 연구하고 `curl_cffi` 라이브러리를 통합하여 SSL 인증서 검증을 우회하는 사용자 지정 세션을 생성함으로써 제한된 환경에서도 데이터에 접근할 수 있도록 했습니다.
